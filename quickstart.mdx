## For teachers

Set up your classroom and create your first assignment in four steps.

<Steps>
  <Step title="Create your account">
    Sign up at the Leopard web dashboard and select the teacher role. Your account will be automatically configured with teacher permissions.
    
    ```typescript
    // Users table includes role-based access
    users: defineTable({
      uid: v.string(),
      role: v.union(
        v.literal("admin"),
        v.literal("student"),
        v.literal("teacher")
      )
    })
    ```
  </Step>
  
  <Step title="Create a classroom">
    Navigate to `/teacher/classroom/new` and provide a classroom name. The system creates a classroom record with you as the owner.
    
    ```typescript
    const createClassroom = useMutation(api.web.teacher.createClassroom);
    
    const classroomId = await createClassroom({
      className: "Intro to Systems"
    });
    ```
    
    <Info>
    Students can see available classrooms and enroll themselves. You can also add assistant instructors using the `assistantIds` array.
    </Info>
  </Step>
  
  <Step title="Create an assignment">
    From your classroom page, click "Create Assignment" to set up new work for students. Configure:
    
    - Assignment name (e.g., "Week 3 - Sorting Algorithms")
    - Description with instructions (supports markdown)
    - Release date (when students can start)
    - Due date (submission deadline)
    
    ```typescript
    const assignmentId = await createAssignment({
      classroomId,
      name: "Week 3 - Sorting Algorithms",
      description: "Implement quicksort and mergesort...",
      releaseDate: Date.parse("2026-03-10T09:00"),
      dueDate: Date.parse("2026-03-17T23:59")
    });
    ```
    
    <Note>
    The due date must be after the release date. Both dates use timestamp values (milliseconds since epoch).
    </Note>
  </Step>
  
  <Step title="Review submissions">
    Once students submit their work, access submissions from `/teacher/classroom/{id}/assignment/{assignmentId}`. You can:
    
    - View the complete list of submissions
    - Replay each student's coding history
    - Add grades and feedback
    - Track submission timestamps
    
    ```typescript
    submissions: defineTable({
      assignmentId: v.id("assignments"),
      studentId: v.string(),
      workspaceId: v.optional(v.id("workspaces")),
      submittedAt: v.number(),
      grade: v.optional(v.float64()),
      submissionFeedback: v.optional(v.string()),
      gradedBy: v.optional(v.string()),
      gradedAt: v.optional(v.number()),
      gradesReleased: v.boolean()
    })
    ```
  </Step>
</Steps>

### Reviewing with history replay

The replay feature at `/replay` reconstructs student work by sequentially applying all captured events. This allows you to:

- Watch code being written in real-time (or accelerated)
- Identify suspicious patterns like large paste operations
- Understand the student's problem-solving process
- Verify authentic work completion

## For students

Start tracking your work and submit assignments in four steps.

<Steps>
  <Step title="Install the VSCode extension">
    Download and install the Leopard VSCode extension from the marketplace or load it manually:
    
    1. Build the extension: `pnpm run build` in `apps/vscode-extension`
    2. Install the generated `.vsix` file in VSCode
    3. The extension activates automatically on startup
    
    ```json
    {
      "activationEvents": ["onStartupFinished"],
      "contributes": {
        "configuration": {
          "properties": {
            "leopard.convexUrl": {
              "type": "string",
              "default": "https://optimistic-kangaroo-331.convex.cloud"
            }
          }
        }
      }
    }
    ```
    
    <Info>
    Check the "Leopard Event Logger" output channel in VSCode to confirm the extension is active and sending events.
    </Info>
  </Step>
  
  <Step title="Join a classroom">
    Navigate to `/student` in the web dashboard to see available classrooms. Click "Enroll" to join:
    
    ```typescript
    const enroll = useMutation(api.web.classroom.enroll);
    
    await enroll({ classroomId });
    ```
    
    Once enrolled, you'll see all assignments for that classroom.
  </Step>
  
  <Step title="Work on your assignment">
    From the classroom page at `/student/classroom/{id}`, launch a workspace for your assignment:
    
    1. Click "Launch Workspace" on the assignment card
    2. Open your project in VSCode with the extension installed
    3. Start coding - all changes are automatically tracked
    
    The extension batches events and sends them every 1000ms by default:
    
    ```typescript
    // Events are debounced and batched for efficiency
    this.debouncer = setTimeout(() => {
      void this.submitEvents();
    }, this.debounceDelay);
    ```
    
    <Note>
    The extension tracks `onDidChangeTextDocument` events with file paths, line ranges, and text changes. Make sure the extension is active while working on your assignment.
    </Note>
  </Step>
  
  <Step title="Submit your work">
    When finished, submit your assignment from the web dashboard:
    
    1. Navigate to the assignment detail page
    2. Click "Submit" to link your workspace to the assignment
    3. The system captures all events from your workspace during the assignment period
    
    Your submission will include:
    
    - Submission timestamp
    - Linked workspace ID with all tracked events
    - Complete coding history for teacher review
    
    ```typescript
    // Submission links your workspace to the assignment
    {
      assignmentId: assignment._id,
      studentId: currentUser.uid,
      workspaceId: workspace._id,
      submittedAt: Date.now(),
      gradesReleased: false
    }
    ```
  </Step>
</Steps>

## Verification

Confirm everything is working:

<AccordionGroup>
  <Accordion title="Extension is tracking">
    Open the VSCode Output panel (View → Output) and select "Leopard Event Logger" from the dropdown. You should see:
    
    ```
    [INIT] Leopard extension activated
    [INIT] Convex URL: https://optimistic-kangaroo-331.convex.cloud
    [INIT] Hostname: your-hostname
    [1234567890] Logged an event: DID_CHANGE_TEXT_DOCUMENT
    [1234567890] SENDING: 5 batched event(s) to Convex...
    [1234567891] SUCCESS: 5 batched event(s) uploaded to Convex
    ```
  </Accordion>
  
  <Accordion title="Workspace is active">
    In the web dashboard, verify your workspace shows as active:
    
    ```typescript
    workspaces: defineTable({
      coderWorkspaceId: v.string(),
      isActive: v.boolean(),
      userId: v.string()
    })
    ```
    
    The `isActive` flag should be `true` while you're working.
  </Accordion>
  
  <Accordion title="Events are stored">
    Teachers can verify events are being captured by accessing the replay page. Events should appear with:
    
    - Correct timestamps in ascending order
    - File paths matching your project structure
    - Text changes reflecting your edits
    
    ```typescript
    events: defineTable({
      eventType: v.string(),
      timestamp: v.number(),
      workspaceId: v.id("workspaces"),
      metadata: v.record(v.string(), v.any())
    })
    ```
  </Accordion>
</AccordionGroup>
